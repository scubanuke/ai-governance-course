# Branch — OT Track

## Learning Objectives

After this module, a student will be able to:

- Derive both design bases — FC-DB and AI-DB — for a facility as engineering artifacts.
- Apply TAF tiering, IEEE V&V, and 62443 mapping.
- Place the Bright Line and Command Broker in an actual operational architecture.
- Explain Local Sovereignty through the Aurora Disruptor case.


For the learner who wants to **produce** a design basis or assessment. Design-basis derivation in depth plus the assessment machinery. Ends by pointing into the capstone with an OT deliverable.

## Units, in order

1. `00-design-basis-derivation.md` — derivation in depth
2. `01-taf-tiering.md` — Tiered Assessment Framework
3. `02-ieee-vnv.md` — IEEE V&V framework
4. `03-62443-mapping.md` — mapping to ISA/IEC 62443
5. `04-command-broker-bright-line-architecture.md` — primitives at architecture level
6. `05-local-sovereignty-aurora-disruptor.md` — the canonical case
