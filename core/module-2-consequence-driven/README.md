# Module 2 — Consequence-Driven Thinking

## Learning Objectives

After this module, a student will be able to:

- Contrast vulnerability-first and consequence-first reasoning.
- Trace the CCE / CIE lineage and explain its engineering provenance.
- Reframe a security problem in their own sector in consequence-first terms.


The mental move everything downstream depends on: from vulnerability-first to consequence-first. Traces the CCE / CIE lineage.

## Units, in order

1. `00-vulnerability-vs-consequence.md` — the inversion
2. `01-cce-cie-lineage.md` — where consequence-driven engineering came from
3. `in-your-sector.md` — closing reflection
